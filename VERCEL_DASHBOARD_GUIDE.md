# 🎯 Что проверить в Vercel Dashboard

## Шаг 1: Найти ваш проект

1. Зайти на [vercel.com/dashboard](https://vercel.com/dashboard)
2. Найти проект (скорее всего он называется как ваш GitHub репозиторий)
3. Кликнуть на проект

## Шаг 2: Проверить настройки проекта

### 2.1 Root Directory (КРИТИЧНО!)

1. **Settings** → **General** → прокрутить вниз
2. Найти **Root Directory**
3. **Должно быть:** `bot` (для bot проекта)

**Если не настроено:**
- Нажать **Edit**
- Ввести: `bot`
- Сохранить
- Vercel автоматически передеплоит

### 2.2 GitHub Branch

1. **Settings** → **Git**
2. Проверить что подключена правильная ветка
3. **Production Branch:** должна быть `vercel-export` или `main`

## Шаг 3: Проверить деплои

1. Вкладка **Deployments** (главная страница проекта)
2. Посмотреть последний деплой

### Статусы:
- ✅ **Ready** (зеленый) = успешно
- ⚠️ **Error** (красный) = есть ошибки
- 🔄 **Building** = идет сборка
- ⏸️ **Queued** = в очереди

### Если деплой с ошибкой:
1. Кликнуть на деплой
2. Открыть вкладку **Build Logs**
3. Прокрутить вниз - там будут ошибки
4. Скопировать ошибку

## Шаг 4: Проверить Functions

1. Вкладка **Functions** (в меню слева)
2. Должны быть видны функции из `bot/api/`

### Если функций нет:
- Проверьте что файлы закоммичены и запушены
- Проверьте Root Directory (должно быть `bot`)
- Проверьте что `vercel.json` в правильном месте

### Если функции есть:
- Кликнуть на функцию (например, `health-check`)
- Открыть вкладку **Logs**
- Можно вызвать функцию и посмотреть логи

## Шаг 5: Проверить Environment Variables

1. **Settings** → **Environment Variables**
2. Проверить наличие переменных

### Обязательные переменные для Bot:

```
SUPABASE_URL
SUPABASE_SERVICE_ROLE_KEY
BOT_TOKEN
GEMINI_API_KEY
JWT_SECRET
REFRESH_SECRET
ALLOWED_TMA_ORIGIN
ALLOWED_WEB_ORIGIN
```

### Если переменных нет:
1. Нажать **Add New**
2. Добавить переменную
3. Выбрать окружения: **Production**, **Preview**, **Development**
4. Сохранить
5. Передеплоить проект (Settings → Redeploy)

## Шаг 6: Тестировать функцию

### Через Dashboard:
1. **Functions** → выбрать функцию (например, `health-check`)
2. Вкладка **Invoke**
3. Нажать **Test Function**
4. Посмотреть результат

### Через браузер/curl:
```bash
# Замените YOUR_PROJECT на ваш проект
curl https://YOUR_PROJECT.vercel.app/api/health
```

Должен вернуться JSON с данными о здоровье системы.

## Шаг 7: Проверить логи

1. **Functions** → выбрать функцию
2. Вкладка **Logs**
3. Вызвать функцию (через браузер или curl)
4. Логи появятся в реальном времени

### Что искать в логах:
- ✅ Успешные запросы
- ❌ Ошибки (красным)
- ⚠️ Предупреждения

## Типичные проблемы в Dashboard

### Проблема 1: "No deployments found"

**Причина:** Проект не подключен к GitHub или ветка не настроена

**Решение:**
1. Settings → Git → Connect Git Repository
2. Выбрать репозиторий
3. Выбрать ветку `vercel-export`
4. Настроить Root Directory: `bot`

### Проблема 2: "Build failed"

**Причина:** Ошибка при сборке

**Решение:**
1. Открыть деплой с ошибкой
2. Build Logs → найти ошибку
3. Чаще всего:
   - Нет файлов (не закоммичены)
   - Неправильный Root Directory
   - Ошибка в коде

### Проблема 3: "Function not found" (404)

**Причина:** Rewrites не настроены или функция не существует

**Решение:**
1. Проверить `vercel.json` rewrites
2. Проверить что функция в `bot/api/`
3. Проверить имя файла

### Проблема 4: "Environment variable not found"

**Причина:** Переменная не добавлена

**Решение:**
1. Settings → Environment Variables
2. Добавить переменную
3. Передеплоить проект

## Быстрая диагностика

### Чеклист:
- [ ] Проект создан в Vercel
- [ ] Root Directory = `bot`
- [ ] Ветка подключена (`vercel-export`)
- [ ] Есть хотя бы один деплой
- [ ] Environment Variables добавлены
- [ ] Functions видны в списке
- [ ] Можно вызвать `/api/health`

## Что делать если что-то не работает

1. **Скопировать ошибку** из Build Logs или Function Logs
2. **Проверить чеклист** выше
3. **Запустить локально:**
   ```bash
   cd bot
   vercel dev
   ```
4. **Проверить файлы:**
   ```bash
   node scripts/check-vercel-ready.js
   ```

## Следующие шаги после успешного деплоя

1. ✅ Протестировать `/api/health`
2. ✅ Протестировать другие endpoints
3. ✅ Обновить `WEBHOOK_URL` для бота
4. ✅ Создать проекты для `web/` и `tma/`
5. ✅ Обновить `VITE_API_BASE_URL` в web/tma
