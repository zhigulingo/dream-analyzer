# 🔍 Диагностика проекта в Vercel

## Шаг 1: Проверить статус деплоя

1. Зайти в [Vercel Dashboard](https://vercel.com/dashboard)
2. Выбрать ваш проект (`dream-analyzer-bot` или как вы назвали)
3. Проверить вкладку **Deployments**

### Что смотреть:
- ✅ **Status:** Ready (зеленый) = успешно
- ⚠️ **Status:** Error (красный) = есть ошибки
- 🔄 **Status:** Building = идет сборка

## Шаг 2: Проверить логи сборки (Build Logs)

Если деплой не прошел:

1. Кликнуть на деплой с ошибкой
2. Открыть вкладку **Build Logs**
3. Прокрутить вниз до ошибок

### Типичные ошибки:

#### ❌ "Cannot find module"
```
Error: Cannot find module './_netlify-adapter'
```
**Решение:** Убедитесь, что `bot/api/_netlify-adapter.js` существует и закоммичен

#### ❌ "Cannot find module '../functions/...'"
```
Error: Cannot find module '../functions/health-check'
```
**Решение:** Обертки ссылаются на `../functions/`, но нужно на `./` или абсолютный путь

#### ❌ "No functions found"
```
No Serverless Functions found
```
**Решение:** Проверьте, что файлы в `bot/api/` и `vercel.json` настроен правильно

## Шаг 3: Проверить структуру проекта в Vercel

1. В проекте → **Settings** → **General**
2. Проверить **Root Directory:**
   - Должно быть: `bot` (для bot проекта)
   - Должно быть: `web` (для web проекта)
   - Должно быть: `tma` (для tma проекта)

### Если Root Directory не настроен:
1. Нажать **Edit**
2. Указать правильную директорию
3. Сохранить
4. Передеплоить

## Шаг 4: Проверить Functions

1. В проекте → **Functions** (вкладка)
2. Должны быть видны все функции из `bot/api/`

### Если функций нет:
- Проверьте, что файлы в `bot/api/` закоммичены
- Проверьте `vercel.json` - правильный ли путь `api/**/*.js`

## Шаг 5: Проверить Environment Variables

1. В проекте → **Settings** → **Environment Variables**
2. Проверить наличие всех переменных:

### Обязательные для Bot:
- `SUPABASE_URL`
- `SUPABASE_SERVICE_ROLE_KEY`
- `BOT_TOKEN`
- `GEMINI_API_KEY`
- `JWT_SECRET`
- `REFRESH_SECRET`
- `ALLOWED_TMA_ORIGIN`
- `ALLOWED_WEB_ORIGIN`

### Если переменных нет:
1. Добавить вручную
2. Или импортировать из Netlify (если есть доступ)

## Шаг 6: Тестировать функцию локально

Установить Vercel CLI и протестировать:

```bash
# Установить Vercel CLI
npm i -g vercel

# Перейти в директорию bot
cd bot

# Запустить локально
vercel dev
```

Откроется локальный сервер, можно тестировать функции.

## Шаг 7: Проверить логи функций

1. В проекте → **Functions** → выбрать функцию
2. Открыть вкладку **Logs**
3. Вызвать функцию через браузер или curl
4. Посмотреть логи

### Тестовая команда:
```bash
curl https://your-project.vercel.app/api/health
```

## Шаг 8: Чеклист диагностики

- [ ] Проект создан в Vercel

- [ ] Ветка `vercel-export` подключена
- [ ] Деплой запустился (хотя бы один)

### ✅ Файлы:

- [ ] `bot/api/health-check.js` существует

### ✅ Конфигурация:

- [ ] Runtime указан: `nodejs20.x`
- [ ] Environment Variables добавлены

- [ ] Функции видны в Vercel Dashboard → Functions
- [ ] Можно вызвать `/api/health` и получить ответ
- [ ] Логи показывают работу (не ошибки)

## Типичные проблемы и решения

### Проблема 1: "Function not found"

**Симптом:** 404 при вызове функции

**Решение:**
1. Проверить `vercel.json` rewrites
2. Убедиться, что функция в `bot/api/`
3. Проверить имя файла (должно совпадать с destination)

### Проблема 2: "Module not found"

**Симптом:** Ошибка в логах "Cannot find module"

**Решение:**
1. Проверить пути в обертках (должны быть `../functions/...`)
2. Убедиться, что `shared/` скопирована
3. Проверить, что все файлы закоммичены

### Проблема 3: "Environment variable not found"

**Симптом:** `process.env.XXX is undefined`

**Решение:**
1. Добавить переменную в Vercel Dashboard
2. Передеплоить проект
3. Проверить, что переменная добавлена для правильного окружения (Production/Preview/Development)

### Проблема 4: CORS ошибки

**Симптом:** CORS error в браузере

**Решение:**
1. Проверить `ALLOWED_TMA_ORIGIN` и `ALLOWED_WEB_ORIGIN`
2. Убедиться, что origins без trailing slash
3. Проверить CORS middleware в функциях

### Проблема 5: Функция работает, но возвращает ошибку

**Симптом:** 200 OK, но в body ошибка

**Решение:**
1. Проверить логи функции в Vercel Dashboard
2. Проверить, что адаптер правильно конвертирует запросы
3. Проверить, что оригинальная функция работает на Netlify

## Быстрая диагностика через терминал

```bash
# Проверить структуру
ls -la bot/api/ | head -20

# Проверить что обертки созданы
ls bot/api/*.js | wc -l

# Проверить vercel.json
cat bot/vercel.json

# Проверить адаптер
head -20 bot/api/_netlify-adapter.js
```

## Что делать дальше

1. **Если деплой успешен:**
   - Протестировать `/api/health`
   - Проверить другие endpoints
   - Начать миграцию функций

2. **Если есть ошибки:**
   - Скопировать ошибку из Build Logs
   - Проверить по чеклисту выше
   - Исправить и передеплоить

3. **Если все работает:**
   - Обновить `WEBHOOK_URL` для бота
   - Обновить `VITE_API_BASE_URL` в web/tma проектах
   - Начать постепенную миграцию функций

---

**Совет:** Начните с простой функции (`health-check`), убедитесь что она работает, затем переходите к более сложным.
